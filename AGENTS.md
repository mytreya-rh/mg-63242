# must-gather-operator

Kubernetes operator that automates OpenShift diagnostic collection (`must-gather`) and uploads results to Red Hat Support cases. Watches `MustGather` custom resources and manages the full lifecycle: gather → upload → cleanup.

## Tech Stack

Go · Kubernetes controller-runtime · operator-sdk · OpenShift

## Key Directories

| Path | Purpose |
|------|---------|
| `api/v1alpha1/` | MustGather CRD types — spec/status structs (`mustgather_types.go`) |
| `controllers/` | Reconciliation logic — core operator behavior |
| `deploy/` | Kubernetes manifests: CRD, ClusterRole, Deployment |
| `config/` | Kustomize-based manifest generation |
| `test/e2e/` | End-to-end tests (Ginkgo harness) |
| `boilerplate/` | Shared build/test infra — **never edit manually** |

## Commands

```bash
make go-build      # build operator binary → build/_output/bin/
make go-test       # unit tests
make validate      # check generated code + boilerplate freeze
make go-check      # lint
make generate      # regenerate CRDs and deepcopy code
```

For CI-equivalent local runs, wrap any target:
```bash
make container-<target>
# or: ./boilerplate/_lib/container-make <target>
```

## Running Locally

```bash
oc apply -f deploy/crds/managed.openshift.io_mustgathers_crd.yaml
oc new-project must-gather-operator
export DEFAULT_MUST_GATHER_IMAGE='quay.io/openshift/origin-must-gather:latest'
OPERATOR_NAME=must-gather-operator operator-sdk run --verbose --local --namespace ''
```

## Critical: Boilerplate System

`boilerplate/` is synced from [github.com/openshift/boilerplate](https://github.com/openshift/boilerplate). A freeze-check runs in CI — **any manual edits will break validation**.

- To update boilerplate: `make boilerplate-update` then commit the result
- The root `Makefile` should only `include boilerplate/generated-includes.mk`
- Subscribed conventions: `openshift/golang-osd-operator` (build/test/lint) and `openshift/golang-osd-operator-osde2e` (E2E)

## Agent Rules

### Never edit generated files
These files are auto-generated — edits will be overwritten and will break `make validate`:
- `api/v1alpha1/zz_generated.deepcopy.go` — regenerated by `make generate`
- `api/v1alpha1/zz_generated.openapi.go` — regenerated by `make generate`
- `deploy/crds/managed.openshift.io_mustgathers.yaml` — regenerated from types
- Anything under `boilerplate/` — updated only via `make boilerplate-update`

### After changing API types
Any edit to `api/v1alpha1/mustgather_types.go` requires regenerating derived artifacts:
```bash
make generate   # updates zz_generated.* and CRD YAML
make validate   # confirms nothing is stale
```

### Controller reconciliation conventions
- Follow the existing pattern in `controllers/mustgather/mustgather_controller.go`: fetch → initialize → finalize → job management
- Initialization (`IsInitialized`) sets defaults (e.g., `ServiceAccountRef.Name` → `"default"`) and requeues if it mutates the object — preserve this pattern
- Deletion cleanup must remove the associated `Secret` and `batchv1.Job` before removing the finalizer — don't skip steps
- The controller owns `batchv1.Job` resources; use `SetupWithManager` watches rather than ad-hoc polling

### Status and conditions
- Use standard Kubernetes condition fields: `type`, `status` (True/False/Unknown), `reason`, `message`, `lastTransitionTime`, `observedGeneration`
- Update `MustGatherStatus.completed` (bool) and `MustGatherStatus.status` (string) to reflect job state
- Never set `completed: true` unless the underlying `batchv1.Job` has actually finished

### Error formatting
Always format errors as descriptive strings with context — never return a bare `err` from a helper function:

```go
// Wrap an upstream error with context
return fmt.Errorf("unable to get clusterversion %q: %w", name, err)

// New error with no upstream cause
return goerror.New("OPERATOR_IMAGE environment variable not found")
```

- Use `%w` (not `%v`) when wrapping so callers can use `errors.Is` / `errors.As`
- Log before returning in helpers: `log.Error(err, "short description", "key", value)`
- Inside `Reconcile`, use `r.ManageError(ctx, instance, err)` (not bare `return reconcile.Result{}, err`) when the error should be reflected in CR status

### Bug fixes must include a regression test
When fixing a bug, add a test case that exercises the exact broken scenario before touching the fix, confirm it fails, then confirm it passes after. The test name should make the regression obvious:

```go
// Good: name describes the bug scenario, not just the function
func TestReconcile_RequeuesWhenInitializationFails(t *testing.T) { ... }
```

- Unit tests go alongside the code under test (e.g., `controllers/mustgather/mustgather_controller_test.go`)
- Cover all branches touched by the change — if the bug was in an `if` block, also verify the `else` path still behaves correctly, adding a test for it if one doesn't already exist
- Run `make go-test` to confirm the suite passes before and after

### Before submitting changes
```bash
make go-test    # unit tests must pass
make validate   # generated code and boilerplate freeze check
make go-check   # lint
```
CI will fail if any of these fail — run them locally first.
